# A/B Testing — Missing Duration & Winner Declaration Fix
## Summary of all changes required

---

## Problem
The A/B test system had no mechanism to:
1. Configure how long the test should run
2. Automatically declare a winner when time expires
3. Auto-send the winning variant to remaining (non-tested) subscribers
4. A `/complete` API endpoint that the frontend was already calling but didn't exist

---

## Files to change

### 1. `backend/models/ab_test.py`
Add two new fields to `ABTestSchema`:

```python
test_duration_hours: Optional[int] = 24   # 1–168 hours
auto_send_winner: bool = True             # send remaining after winner declared
```

### 3. `backend/tasks/ab_testing.py`

**Add two new Celery tasks** (from `tasks_ab_testing_additions.py`):

**`check_ab_test_expiry`** (periodic, every 15 min):
- Queries all `status="running"` A/B tests
- Checks `start_date + test_duration_hours` vs `now`
- Calls `auto_complete_ab_test.delay()` for expired tests

**`auto_complete_ab_test`** (async worker):
- Calculates winner using `calculate_test_results_sync()`
- Marks test `status="completed"` with `winner_variant`
- Calls `apply_winner_to_campaign_sync()` if `apply_to_campaign=True`

## Flow after fix

```
Create A/B test
  └─ test_duration_hours = 24, auto_send_winner = true

Start test (sends variant A/B emails to sample)
  └─ status = "running", start_date = now

Celery Beat (every 15 min) → check_ab_test_expiry
  └─ elapsed >= test_duration_hours?
       └─ YES → auto_complete_ab_test.delay()
                 ├─ calculate winner
                 ├─ status = "completed"
                 └─ auto_send_winner=true?
                      └─ apply_winner_to_campaign_sync()
                           ├─ update campaign subject/content
                           └─ send_campaign_batch.delay()  ← sends to remaining subscribers

OR manual: POST /ab-tests/{id}/complete → same logic
```
