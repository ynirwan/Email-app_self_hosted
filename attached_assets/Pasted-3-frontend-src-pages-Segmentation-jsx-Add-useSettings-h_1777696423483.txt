9. frontend/src/components/AuditTrail.jsx
This file has two versions in your codebase (the older one and the refactored one). Both have the same problem.

9a — fmtTime / formatTimestamp local helpers
Both versions define a local date formatter that falls back to .toLocaleString('en-US', ...). These ignore the user's timezone for the absolute-date fallback (when the log is > 7 days old).

The relative portions (3m ago, 2h ago) are timezone-independent — they're fine. Only the absolute date fallback needs to change.

js
// BEFORE (in both versions — the fallback at the end)
return date.toLocaleString('en-US', {
  month: 'short', day: 'numeric',
  year: date.getFullYear() !== now.getFullYear() ? 'numeric' : undefined,
  hour: '2-digit', minute: '2-digit'
});
Because AuditTrail is a component (not a page), it can call useSettings() directly. Add at the top of the component:

jsx
const { formatDateTime } = useSettings();
Then rewrite the helper as a closure or inline it:

jsx
// AFTER — define inside the component so formatDateTime is in scope
const fmtTime = (timestamp) => {
  const date = new Date(timestamp);
  const diffMs = Date.now() - date;
  const mins  = Math.floor(diffMs / 60000);
  const hours = Math.floor(diffMs / 3600000);
  const days  = Math.floor(diffMs / 86400000);
  if (mins  < 1)  return 'Just now';
  if (mins  < 60) return `${mins}m ago`;
  if (hours < 24) return `${hours}h ago`;
  if (days  < 7)  return `${days}d ago`;
  return formatDateTime(timestamp);   // ← user's timezone
};
