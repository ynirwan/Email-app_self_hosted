# Implementation Guide: User Timezone Support

To implement consistent timezone support across the platform based on the user's saved preference, follow these steps.

## 1. Frontend Dependencies
Install `date-fns-tz` or `luxon` for easy timezone handling.
Command: `npm install date-fns-tz`

## 2. Global Utility (frontend/src/utils/date.js)
Create a utility to format dates using the user's timezone:
```javascript
import { formatInTimeZone } from 'date-fns-tz';

export const formatUserDate = (date, timezone = 'UTC', formatStr = 'yyyy-MM-dd HH:mm:ss') => {
  if (!date) return '-';
  try {
    return formatInTimeZone(new Date(date), timezone, formatStr);
  } catch (err) {
    return date;
  }
};
```

## 3. Frontend Refactoring
In components that display dates (e.g., Campaigns, Analytics, Audit Logs), fetch the user's timezone from the auth context/state and use the utility.

Example (Campaigns.jsx):
```javascript
import { formatUserDate } from '../utils/date';
// ... inside component
<Td>{formatUserDate(c.created_at, userTimezone)}</Td>
```

## 4. Scheduled Campaigns (frontend/src/pages/Campaigns.jsx)
When scheduling a campaign, the date/time picker usually gives local time. You must convert this to UTC before sending it to the backend.

```javascript
import { zonedTimeToUtc } from 'date-fns-tz';

const confirmSchedule = async () => {
  const localDateTime = `${scheduleDate}T${scheduleTime}`;
  const utcDate = zonedTimeToUtc(localDateTime, userTimezone).toISOString();
  await API.post(`/campaigns/${id}/schedule`, { scheduled_time: utcDate });
};
```

## 5. Backend (backend/routes/campaigns.py)
Ensure all scheduling logic compares incoming `scheduled_time` (which should now be UTC) against `datetime.utcnow()`. Since the backend already uses UTC, no major change is needed if the frontend sends UTC.

## 6. Files to Update
- `frontend/src/pages/Campaigns.jsx`: Format display dates and convert scheduling inputs.
- `frontend/src/pages/Dashboard.jsx`: Format chart/stat dates.
- `frontend/src/components/AuditTrail.jsx`: Format event timestamps.
- `frontend/src/pages/SubscriberListView.jsx`: Format subscriber addition dates.
- `backend/tasks/campaign/email_campaign_tasks.py`: Ensure logs and metrics remain in UTC.
