# Implementation Guide: Multi-language Support (i18n)

To implement full multi-language support across the Email Marketing Platform, follow these steps.

## 1. Frontend Dependencies
Install the following packages in the `frontend` directory:
- i18next
- react-i18next
- i18next-browser-languagedetector
- i18next-http-backend

Command: `npm install i18next react-i18next i18next-browser-languagedetector i18next-http-backend`

## 2. Configuration (frontend/src/i18n.js)
Create this file to initialize i18next:
```javascript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import Backend from 'i18next-http-backend';

i18n
  .use(Backend)
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: 'en',
    debug: false,
    interpolation: {
      escapeValue: false,
    },
    backend: {
      loadPath: '/locales/{{lng}}.json',
    }
  });

export default i18n;
```

## 3. Translation Files (frontend/public/locales/)
Create JSON files for each supported language in `frontend/public/locales/`:
- en.json
- es.json
- fr.json
- de.json
- zh.json
- hi.json

Example (en.json):
```json
{
  "dashboard": "Dashboard",
  "campaigns": "Campaigns",
  "subscribers": "Subscribers",
  "settings": "Settings",
  "profile_info": "Profile Information"
}
```

## 4. Main Entry (frontend/src/main.jsx)
Import the i18n config at the top:
`import './i18n';`

## 5. Integrating User Preference (frontend/src/App.jsx)
In your main App component, fetch the user's saved language preference from the backend and set i18next accordingly:

```javascript
import { useTranslation } from 'react-i18next';
// inside component...
const { i18n } = useTranslation();

useEffect(() => {
  if (userData?.language) {
    i18n.changeLanguage(userData.language);
  }
}, [userData, i18n]);
```

## 6. Refactoring Components
Replace hardcoded text with the translation hook.
Example:
```javascript
import { useTranslation } from 'react-i18next';

function Sidebar() {
  const { t } = useTranslation();
  return (
    <div>{t('dashboard')}</div>
  );
}
```

## 7. Files to Update
- `frontend/src/components/Sidebar.jsx`: Localize navigation links.
- `frontend/src/pages/Dashboard.jsx`: Localize stats and charts.
- `frontend/src/pages/Campaigns.jsx`: Localize table headers and buttons.
- `frontend/src/pages/UserSettings.jsx`: Localize form labels.
- `frontend/src/components/Layout.jsx`: Localize general UI elements.

## 8. Backend Considerations
The backend already supports saving and retrieving the `language` preference in the `/api/auth/me` endpoints. Ensure that when a user updates their language in `UserSettings.jsx`, you also trigger `i18n.changeLanguage()` in the frontend state.
